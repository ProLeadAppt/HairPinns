SEO Audit — Hair Pinns

Full findings across all four areas. Each issue is rated by impact.

🔴 Critical — Root Cause of Most Problems

Issue #1: SPA with no pre-rendering

The site is a client-side React SPA. When Google, Facebook, or any crawler fetches a URL, they receive this:

<body>
  <div id="root"></div>  ← empty
  <script src="/assets/js/index-abc123.js"></script>
</body>

No title. No description. No OG tags. No content. Googlebot may eventually execute JS, but social crawlers (Facebook, LinkedIn, Slack) never do — so every shared link shows a blank preview. This is the root cause of issues #2 and #3.

🔴 Critical — Social Previews

Issue #2: OG/Twitter tags invisible to social crawlers

All OG meta tags (og:title, og:description, og:image, twitter:card) are injected by react-helmet at runtime. Social crawlers don't run JS, so they never see these tags.

  • Facebook share → blank card
  • LinkedIn share → no preview image
  • Slack/Teams unfurl → just the URL
  • iMessage link preview → nothing

Fix: Inject static OG tags into index.html at build time, or add a pre-rendering layer (SSG/SSR).

🟠 High — Google Indexing

Issue #3: Googlebot sees thin/empty content on first crawl

Googlebot can execute JS but does so in a second wave, often days after discovery. This means:

  • New pages take longer to be indexed
  • Google may cache empty versions and show "Page title" as the URL in results
  • Core Web Vitals measured on empty shell, not real content
  • Suburb/location pages (your most valuable SEO pages) are especially at risk
Issue #4: URL inconsistency — hairpinns.com vs hairpinns.com.au

index.html hardcodes hairpinns.com in schema and sitemap. robots.txt also points to hairpinns.com. But schema.ts, SEOHead, and the business config all use hairpinns.com.au. Google treats these as two different sites — this fragments your authority.

Issue #5: Hardcoded review counts in schema are stale

Multiple places hardcode reviewCount: "53" and "815" — these will drift out of sync with reality. generateLocalBusinessSchema has 53; generateEnhancedServiceSchema has 815; index.html also has 53. Inconsistent counts can cause Google to distrust your schema entirely.

🟠 High — AI Crawlers / AEO

Issue #6: robots.txt missing AI crawler rules

No explicit rules for AI crawlers. To appear in ChatGPT, Perplexity, and Google AI Overviews you need:

User-agent: GPTBot
Allow: /

User-agent: Google-Extended
Allow: /

User-agent: PerplexityBot
Allow: /

User-agent: anthropic-ai
Allow: /

User-agent: ClaudeBot
Allow: /

Without explicit permission, some AI crawlers skip sites to avoid legal risk.

Issue #7: No llms.txt file

llms.txt is an emerging standard (like robots.txt for AI) that tells LLMs what your site is about, what to index, and what to skip. Not having one means AI crawlers have to guess. Adding a well-structured one can significantly improve how AI systems describe your business.

🟡 Medium — Schema / Structured Data

Issue #8: Duplicate/conflicting schemas on homepage

The homepage renders: Organization + LocalBusiness + KnowledgeGraph (another Organization) + Store + WebPage + HowTo + FAQ + Author. That's 8+ schemas. Having two Organization schemas with different @id values on the same page confuses Google's entity understanding.

Issue #9: ProductDetail page uses react-helmet, not SEOHead

ProductDetail.tsx imports Helmet directly, bypassing the centralised SEOHead component. This means it might be missing Twitter cards, hreflang, or canonical enforcement that SEOHead provides consistently.
